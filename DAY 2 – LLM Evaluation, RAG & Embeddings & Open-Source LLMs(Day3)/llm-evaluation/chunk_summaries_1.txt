**How Django Processes a Request**

Here's what happens when you run the Django development server and make requests to Web pages:

1. The `python manage.py runserver` command looks for a file called `settings.py`, which contains the `ROOT_URLCONF` setting.
2. Django loads the URLconf pointed-to by the `ROOT_URLCONF` setting.
3. It checks each URLpattern in the URLconf, comparing the requested URL with the patterns one at a time, until it finds one that matches.
4. When it finds a match, it calls the view function associated with that pattern, passing a `HttpRequest` object as the first parameter.
5. The view function is responsible for returning an `HttpResponse` object.

**Loose Coupling in URLconfs**

Django's URLconfs are an example of loose coupling, where the URL definitions and view functions are loosely coupled, making it easy to switch out one piece without affecting the other. This is in contrast to other Web development platforms that couple the URL to the program.

**Django Template System Basics**

* Django templates separate presentation from data, making it easier to maintain and update designs.
* Templates are strings of text with placeholders and logic tags that regulate how the document is displayed.
* Templates can generate any text-based format, not just HTML.

**Template Structure**

* Variables are surrounded by double curly braces (e.g., `{{ person_name }}`).
* Block tags are surrounded by curly braces and percent signs (e.g., `{% if ordered_warranty %}`).
* Filters are attached to variables using a pipe character (e.g., `{{ ship_date|date:"F j, Y" }}`).

**Using the Template System**

* Create a `Template` object by providing the raw template code as a string or by designating the path to a template file.
* Call the `render()` method of the `Template` object with a given set of variables (context) to evaluate the template and return a fully rendered string.

**Key Concepts**

* Variables: placeholders for data that will be inserted into the template.
* Block tags: logic statements that regulate how the document is displayed.
* Filters: functions that alter the display of variables.
* Context: the set of variables passed to the `render()` method to evaluate the template.

**Creating a Django Template Object**

1. Import the `Template` class from `django.template`.
2. Create a template object by passing a string of template code to the `Template` class constructor.

**Rendering a Template**

1. Create a context object by passing a dictionary of variable names and values to the `Context` class constructor.
2. Call the `render()` method on the template object, passing it the context object.

**Key Facts**

* Variable names must begin with a letter and can contain digits, underscores, and dots.
* Variable names are case sensitive.
* The `render()` method replaces template variables with actual values and executes block tags.
* You can render multiple contexts through the same template object.

**Example**

```python
from django.template import Template, Context

raw_template = """Hello, {{ name }}"""
t = Template(raw_template)
c1 = Context({'name': 'John'})
c2 = Context({'name': 'Julie'})

print(t.render(c1))  # Output: Hello, John
print(t.render(c2))  # Output: Hello, Julie
```
